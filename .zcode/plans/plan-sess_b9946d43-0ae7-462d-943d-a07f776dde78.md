## 排版紧凑化方案

目标：整体字体和排版更紧凑，课程正文字号从 18px 降到 16px，目录更紧凑。改动集中在 5 个文件，全部为数值微调。

### 文件 1: `src/styles/tokens.css`（字号 token，单一真源）
- `--text-body-lg`: `1.125rem` → `1rem`（18px→16px，正文降一档）
- `--text-h1`: `2.5rem` → `2.25rem`（40→36px）
- `--text-h2`: `1.875rem` → `1.625rem`（30→26px）
- `--text-h3`: `1.375rem` → `1.25rem`（22→20px）

### 文件 2: `src/styles/global.css`（行高与标题间距）
- `.prose-lesson` line-height: `1.7` → `1.65`
- `h1`: line-height `1.15`→`1.2`，下 margin `space-6`→`space-5`（24→20px）
- `h2`: line-height `1.25`→`1.3`，margin `space-12/space-4`→`space-10/space-3`（上40→32, 下16→12px）
- `h3`: line-height `1.3`→`1.35`，margin `space-8/space-3`→`space-6/space-2`（上32→24, 下12→8px）
- `.katex-display` margin: `space-6` → `space-5`（24→20px）

### 文件 3: `tailwind.config.mjs`（同步 fontSize 的 lineHeight 硬编码值）
- `body-lg` lineHeight: `1.7` → `1.65`
- `h1` lineHeight: `1.15` → `1.2`
- `h2` lineHeight: `1.25` → `1.3`
- `h3` lineHeight: `1.3` → `1.35`

### 文件 4: `src/components/layout/SidebarNav.astro`（通用侧边栏更紧凑）
- 模块区块 `mb-6` → `mb-4`
- 模块标题 `py-2` → `py-1`
- 链接 `py-1.5` → `py-1`，`text-sm` → `text-[13px]`

### 文件 5: `src/pages/courses/[course]/[...slug].astro`（内联侧边栏更紧凑）
- 课程标题区 `pt-8 pb-4` → `pt-6 pb-3`
- 模块区块 `mb-5` → `mb-3`
- 模块标题 `text-[11px]` → `text-[10px]`
- 链接 `py-1.5` → `py-1`，`text-[13px]` → `text-[12px]`

### 不改动
- 字体栈、颜色、圆角、阴影、动效 token
- 间距 4px 基准刻度
- 内容宽度 `--content-prose-width: 680px`
- KaTeX 相对字号 `1.05em`（随正文自动缩放）

### 验证
改完后运行 `npm run build` 确认无构建错误，并可启动 dev server 目视检查课程页排版的紧凑度。
