# Singular Value Decomposition

Eigenvectors only work for square matrices. But what about non-square matrices — the ones that change dimensions? **SVD** is the ultimate decomposition: it works for *every* matrix, no exceptions.

<MathBlock expression={'A = U \\Sigma V^T'} />

Three pieces:
- **V** (right singular vectors): an orthonormal basis for the input space — the "natural" input directions
- **Σ** (singular values): how much each direction is stretched
- **U** (left singular vectors): an orthonormal basis for the output space — where the stretched directions land

## The geometric story

SVD decomposes any linear transformation into three atomic steps:

1. **Rotate** the input by Vᵀ — align the input with the "natural" directions (the right singular vectors)
2. **Stretch** along those axes by Σ — scale each direction independently by its singular value σ₁, σ₂, ...
3. **Rotate again** by U — align the stretched result with the output axes (the left singular vectors)

The result: a unit circle becomes an ellipse whose semi-axes have lengths σ₁ and σ₂. The longest axis points along the first left singular vector; the input direction that maps to it is the first right singular vector.

## SVD vs. eigendecomposition

| Property | Eigendecomposition | SVD |
|----------|-------------------|-----|
| Works for | Square matrices only | **Any matrix** |
| Bases | One basis (eigenvectors) | Two bases (U and V) |
| Orthogonality | Not guaranteed | **Always orthogonal** |
| Values | Eigenvalues (can be negative/complex) | Singular values (always ≥ 0) |

The key geometric difference: eigendecomposition uses the **same coordinate system** for input and output — you're asking "which directions stay on their own line?" SVD uses **different coordinate systems** for input and output — you're asking "which input direction maps to which output direction, and how much does it stretch?"

This is why SVD always works: you're free to pick the most natural basis for each side independently. The eigendecomposition insists on one basis for both, which is only possible when the matrix is square and well-behaved.

## The condition number

The ratio κ = σ₁/σ₂ is the **condition number** of the matrix. It measures how "elongated" the ellipse is — how much the transformation distorts the input:

- **κ = 1** (circle → circle): The transformation preserves all directions equally. Perfect conditioning.
- **κ ≈ 3**: The ellipse is moderately stretched. Some directions are amplified more than others.
- **κ > 10**: The ellipse is very flat. The transformation massively amplifies one direction while crushing another. This is **ill-conditioning** — small changes in input can cause huge changes in output.
- **κ = ∞** (σ₂ = 0): The ellipse collapses to a line. The matrix is singular — information is lost.

A matrix is invertible if and only if all its singular values are positive. The condition number tells you *how close* a matrix is to being singular.

## Where SVD lives

- **PCA (data science):** The singular vectors of a data matrix are the principal components — the directions of maximum variance in your data
- **Image compression:** Keep only the largest singular values and discard the rest. A 1000×1000 image might be well-approximated by just 50 singular values — a 20x compression
- **Recommendations (Netflix):** Low-rank approximation via SVD reveals hidden preferences in sparse user-movie rating matrices
- **Natural language processing:** Latent semantic analysis applies SVD to word-document matrices, uncovering hidden topics
