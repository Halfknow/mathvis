# Functions as Vectors

Everything we've built — span, basis, dimension, linear transformation — was motivated by arrows in space. But the same algebraic structure appears in places that have nothing to do with arrows.

**Functions are vectors.** You can add them, scale them, and find bases for them. All the theorems still hold.

## The rules of being a vector

A "vector space" is anything where you can:
1. **Add** two elements: f(x) + g(x) is a new function
2. **Scale** by a number: 3*f(x) is a new function
3. **Distribute**: c*(f + g) = c*f + c*g

Polynomials satisfy all of these. So do continuous functions, differentiable functions, and many other families.

## A basis for polynomials

The space of polynomials of degree ≤ 2 has a natural basis:

<MathBlock expression={'\\{1, \\; x, \\; x^2\\}'} />

Any quadratic polynomial is a unique linear combination: p(x) = a + bx + cx². The coordinates are (a, b, c) — just like a point in 3D space, but the "directions" are 1, x, and x² instead of arrows.

Differentiation is a **linear transformation** on this space:

<MathBlock expression={'\\frac{d}{dx}(a + bx + cx^2) = b + 2cx = b \\cdot 1 + 2c \\cdot x'} />

In the basis (1, x, x²), differentiation is the matrix:

<MathBlock expression={'D = \\begin{bmatrix} 0 & 1 & 0 \\\\ 0 & 0 & 2 \\\\ 0 & 0 & 0 \\end{bmatrix}'} />

The null space of D is the constant functions — the polynomials whose derivative is zero. The column space is all polynomials of degree ≤ 1. All the linear algebra you've learned applies.

<FormulaReadout formulas={[
  { label: "多项式基", expression: "\\mathcal{P}_n = \\text{span}(1, x, x^2, \\ldots, x^n)" },
  { label: "线性变换", expression: "\\frac{d}{dx}(af + bg) = a\\frac{df}{dx} + b\\frac{dg}{dx}" },
]} />

<KeyInsight>
Functions are vectors: you add them, scale them, and decompose them into bases. Differentiation, integration, and other operations are linear transformations on function spaces. The same theorems that govern arrows govern functions.
</KeyInsight>
