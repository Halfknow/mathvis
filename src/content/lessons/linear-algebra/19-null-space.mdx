# Null Space

Every transformation sends *some* vectors to the origin. The set of all vectors that get crushed to zero is the **null space** (also called the **kernel**).

<MathBlock expression={'\\text{Null}(A) = \\{\\vec{x} : A\\vec{x} = \\vec{0}\\}'} />

If a matrix is invertible, only the zero vector gets sent to zero — the null space is trivial. But when the transformation collapses space (det = 0), an entire line or plane of inputs disappears into the origin.

## See the collapse

Below, gray dots are input vectors around a circle. Blue lines show where each input gets sent. When det drops to zero, the red line appears — that's the null space. Every vector on that line maps straight to the origin.

<NullSpaceVis client:visible />

Try dragging the matrix toward det = 0 (e.g., make the second row a multiple of the first). Watch the null space line appear as the transformation collapses.

## Why the null space matters

The null space tells you about **information loss**:
- **Null(A) = (0)**: No information is lost. Every input maps to a unique output. The matrix is invertible.
- **Null(A) = a line**: A whole direction of inputs is indistinguishable — they all produce the same output. You've lost one dimension of information.
- **Null(A) = a plane**: Two dimensions of inputs are indistinguishable.

This connects directly to the uniqueness of solutions: if Ax = b has a solution x, and Null(A) contains a nonzero vector n, then x + n is *also* a solution. The null space is the ambiguity in your answer.

## In three dimensions

In 3D, the null space is a line or plane of inputs that all map to zero. The yellow dots mark the null space direction — every point on that line gets crushed to the origin.

<KeyInsight>
The null space is the set of inputs that the transformation annihilates. When it's more than just the zero vector, the transformation has lost information — and solutions to Ax = b are not unique.
</KeyInsight>

**Connections.** The null space is what gets annihilated — the dimensions that don't survive the transformation. Finding the null space by row reduction reveals the 'free variables' — the directions you can move without changing the output. When the null space is non-trivial, solutions to Ax=b are never unique (you can always add a null space vector).
